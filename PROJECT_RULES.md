## CSS

common.cssは基本変更しない。

ページ固有CSSには

- ページ固有レイアウト

- ページ固有装飾

のみを書く。

レイアウトは原則として`display: flex`を優先する。単純な左右配置、均等カラム、折り返しカード、縦積みには`display: grid`を使用しない。行と列を同時に厳密管理する必要がある場合のみGridを検討する。

セクション内の写真をPC可変幅で画面端まで伸ばす場合は、961px〜1430pxで写真幅に外側余白分を加え、同量のマイナスマージンで外へ逃がす。1431px以上は最大コンテンツ幅内に収め、画面端との間に白い余白が出てもよい。

## レスポンシブ

ブレークポイント

960px / 961px

PCデザインの最大幅は1430pxを基準とする。

画面幅に応じて位置関係を維持する必要がある要素（絶対配置のキャラクターや装飾など）は、以下のように指定を分ける。

- 961px〜1430px：`vw`（コンテナ基準が適切な場合は `cqw` も併用）で可変指定
- 1431px以上：最大幅時の位置を `px` で固定

```css
/* 1431px以上を含む基本値 */
@media print, screen and (min-width: 961px) {
    .element {
        top: -96px;
    }
}

/* 961px〜1430pxは画面幅に比例 */
@media screen and (min-width: 961px) and (max-width: 1430px) {
    .element {
        top: -6.7vw;
    }
}
```

単純なサイズ可変だけでなく、基準画像やセクションに対する相対位置が横幅によってズレないかを確認する。

SPで「左側にキャラクター、右側に本文」のように見えるレイアウトでも、キャラクターが本文の横幅を圧迫しないデザインの場合はFlexの横並びにしない。

- 本文は`width: 100%`としてコンテンツ幅をすべて使用する
- キャラクターは`position: absolute`で通常フローから外す
- キャラクターはセクションの余白や画像との重なり部分へ配置する
- 親要素を配置基準にし、本文の改行位置がキャラクターの有無で変わらないようにする

```css
.section_inner {
    position: relative;
}

.section_inner .chara {
    position: absolute;
    top: -28vw;
    left: 0;
}

.section_inner .text {
    width: 100%;
}
```

SPでは、キャプションなど明らかに小さくてよい文字を除き、本文を含む通常テキストの最小サイズを以下とする。

```css
font-size: clamp(calc(3.6rem * 0.375), 3.75vw, 3.6rem);
```

360px幅で本文が12px相当になる指定は使用しない。

必ず

- PC

- 961px

- 960px

- 390px

- 375px

で確認する。
