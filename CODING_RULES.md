# CODING RULES

## 基本方針

- デザインを忠実に再現する。
- 既存ページと同じ役割の部品は既存HTMLを優先して流用する。
- HTML構造（DOM）も可能な限り既存ページと揃える。
- ページ固有の事情がある場合のみ、新しい構造を採用する。
- 新しい構造を採用する場合は理由を説明する。

---

## HTML

- figure / figcaption を適切に使用する。
- 見出しレベルはコンテンツ構造に合わせる。
- 不要なラッパーを作らない。
- 表示・JavaScript・アクセシビリティのいずれにも使われないclassは作らない。
- href="#" を残さない。
- alt は画像の役割に応じて設定する。

---

## class

- 既存classを優先して使用する。
- 同じ役割のclassを新設しない。
- 見た目だけのclassを作らない。
- class名から役割が分かるようにする。

---

## CSS

- common.cssを優先する。
- ページ固有CSSには差分だけを書く。
- 他ページ専用CSSへ依存しない。
- セレクタ範囲は必要最小限にする。
- 汎用classへページ固有デザインを当てない。
- 不要CSSを残さない。

---

## レスポンシブ

- PCの縮小版を作らない。
- 情報の意味を優先してSPを設計する。
- 比較・工程・時系列はSPで縦並びも検討する。
- 375px付近でも判読性を保つ。

---

## デザイン再現

- 支給素材がある場合はそれを使用する。
- CSSによる近似表現は最後の手段とする。
- サイズ・余白・字間・行間・改行位置もデザインに合わせる。

---

## 実装後

実装後は必ずレビューを行う。
